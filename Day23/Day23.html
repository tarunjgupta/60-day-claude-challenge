Section 07
MVP Recommendation
What to build first, what NOT to build, and how to know if it's working.
✅
What to Build (MVP Scope)
  • User auth — Email/Google signup
  • Create/join pod — WhatsApp-style invite link
  • Morning 3-task check-in — Simple text input
  • Evening check-off — Checkbox UI
  • Pod feed — See everyone's tasks & completions
  • Basic Grind Score — Simple calculation (consistency + completion)
  • Pod leaderboard — Real-time ranking within pod
  • 3 daily notifications — Morning reminder, mid-day nudge, evening close-out
  • 1 hardcoded success story/day — Curate 20 stories manually
  • Shareable Grind Score card — Image export for Instagram/LinkedIn
✗
What NOT to Build
  • Activity tracking / screen monitoring — Privacy landmine. Not needed.
  • AI-generated roasts — Removed from concept. Risk > reward.
  • College leaderboard — Needs scale. Add at 500+ users.
  • National leaderboard — Needs scale. Add at 5,000+ users.
  • Payment / premium tier — Validate retention FIRST.
  • AI coaching / career advice — V2 feature. Over-engineering for MVP.
  • Focus timer — Nice-to-have, not core loop.
  • Parent dashboard — After proving B2C first.
  • Hindi/regional language support — English MVP, localize later.
  • iOS app — Android-first. 90%+ of ICP is Android.

Section 08
MoSCoW Prioritization
Ruthless prioritization. If it's not a "Must Have," it doesn't go in the MVP.

🟢 Must Have — Ship Breaks Without These

  • User signup (email/Google OAuth)
  • Create pod + invite link (WhatsApp shareable)
  • Morning 3-task commitment input
  • Evening task check-off
  • Pod feed (see everyone's tasks + completion status)
  • Basic Grind Score calculation
  • Pod leaderboard (rank within pod)
  • Morning check-in notification
  • Streak counter (individual + pod)

🔵 Should Have — Strong Value-Add, Week 2 of Build

  • Shareable Grind Score image card
  • Mid-day competitive nudge notification
  • Evening close-out reminder
  • 1 daily success story (hardcoded)
  • Pod chat (simple messaging)
  • Streak-at-risk alert notification
  • Basic user profile page

🟡 Could Have — Nice Extras, V1.1

  • College leaderboard
  • Detailed Grind Score analytics (weekly breakdown)
  • Multiple pods per user
  • Custom notification schedules
  • Comeback nudge after missed days
  • Auto-match for solo users (public pods)

🔴 Won't Have — Explicitly Out of Scope

  • Activity tracking / screen monitoring
  • AI-generated roast notifications
  • AI career coaching
  • Payment processing / premium tier
  • National leaderboard
  • Parent dashboard
  • iOS app
  • Hindi / regional language support

Section 11
30-Day MVP Plan
From today to a working MVP with 10 beta users — day by day.
W1

Customer Validation (Days 1–7)

Talk to humans before writing code
D1
Run a manual WhatsApp pod for 14 days — 6 friends, morning tasks, evening check-offs. YOU are the product. Observe what works.
D2–3
Conduct 15 user interviews — "Tell me about a day you wasted." "What would make you actually follow through?" Record patterns.
D4
Synthesize interview findings — affinity map. Which pain hurts most? Do students actually want pod accountability?
D5–6
Build landing page (Framer/Carrd) — "5 friends. 3 tasks. 1 Grind Score." Waitlist email signup. Launch on LinkedIn + campus groups.
D7
Post interview insights on LinkedIn — build audience. Share learnings. Target: 50 waitlist signups by Day 7.
W2

MVP Build Sprint (Days 8–14)

Build only Must-Haves from MoSCoW
D8
Setup project — React + Node/Express + MongoDB. Auth (Google OAuth). Basic CI/CD.
D9–10
Build core loop — Pod creation, invite link, morning task input, evening check-off, pod feed showing all members' tasks.
D11–12
Build Grind Score + pod leaderboard — Simple calculation. Real-time ranking. Streak counter for individual + pod.
D13
Add notifications — Morning check-in reminder, evening close-out. Use email (Resend) or push (OneSignal free tier).
D14
Deploy MVP — Vercel (frontend) + Railway/Render (backend). Test end-to-end. Invite 2 friends to verify pod flow works.
W3

Beta Launch (Days 15–21)

10 real users, 2 pods, daily observation
D15
Onboard 10 beta users from waitlist. Form 2 pods. Full access. Daily feedback channel (WhatsApp group).
D16–19
Observe daily — Who checks in? Who skips? What do they say in pod chat? Where do they get stuck? Log everything.
D20
Add Should-Haves — Shareable Grind Score card, mid-day nudge, 1 daily success story (hardcoded).
D21
Measure D7 retention — of 10 beta users, how many are still checking in daily? Target: 4+ users = >40%.
W4

Iterate & Scale Prep (Days 22–30)

Fix, learn, decide
D22–24
Fix top 3 issues from beta feedback. Only 3. No feature creep. Ship within 48 hours.
D25
Run price sensitivity survey — 100+ responses. Test ₹49 / ₹99 / ₹149 / ₹199. Find the cliff.
D26–28
Expand to 50 users — from waitlist + campus outreach. Form 8–10 pods. Measure D7 retention at scale.
D29
Write "30-Day Learnings" LinkedIn post — interview insights, beta data, what worked, what didn't.
D30
GO / NO-GO decision — D7 retention >40% AND waitlist >200? → GO. If not → pivot one element, run another 30-day cycle.

Final Verdict

WEIGHTED AVERAGE · OUT OF 100
🟢 Strong Demand Signal
The customer is clearly defined. The problem is daily and painful. The solution (pod accountability + Grind Score) is differentiated and behaviorally sound. The MVP scope is tight and buildable in 14 days with MERN. Proceed to build after completing the WhatsApp pod experiment and 15 user interviews.

Why 🟢 Strong (Not 🟡 Promising)

  • Problem frequency: daily (not weekly/monthly)
  • Pain intensity: 9/10 — students actively know they're wasting time
  • Cultural fit: India's competitive student culture is native to the pod + leaderboard model
  • Founder IS the ICP — rare authenticity advantage
  • No direct competitor in India-first social accountability
  • Multiple revenue paths (B2C + B2B) reduce binary risk

What Could Downgrade to 🟡

  • If D7 retention <25% — core loop isn't compelling
  • If <40% of signups form pods — pod model is friction-heavy
  • If waitlist <100 in 14 days — demand signal is weak
  • If interviews reveal students don't actually want peer visibility
  • If WhatsApp pod experiment dies within 5 days
  • If pricing survey shows cliff below ₹49/month
